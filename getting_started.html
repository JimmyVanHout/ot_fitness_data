About

This site analyzes the data from your OrangeTheory Fitness workout summary emails (emails from otbeatreport@orangetheoryfitness.com) and displays the results. Note that this site is not affiliated with OrangeTheory Fitness. Highlights:

  • It runs entirely in your browser, so your data stays on your computer--it is not sent to a server.

  • The program that reads the email data currently only works with Gmail email accounts.

How it works: Whenever you want to see your newest data on this site, you will:

  1. Run a program to extract the data from the workout summary emails.

  2. Upload the data to this site.

You will only have to complete the sequence of steps in the First Time section once. After that, running the program is much simpler and faster (see Next Time), and if you use Mac or Linux it is as simple as double-clicking a script you can save to your desktop.

First Time

The first time you want to run the data-extracting program and upload your data, you will be:

  • Creating a new Gmail account to use only for your workout summary emails

  • Forwarding past emails from your usual account to the new account using the data-extracting program (currently only works to forward from Gmail accounts)

  • Forwarding future emails using a Gmail filter set up on your usual account

  • Running the program

  • Uploading the newly created data file to this site, which will analyze and display the data for you

The program extracts the workout data from the emails to a CSV file. For a more detailed overview of how the program works, read the README file (the next several steps are from that document). Important to note:

  1. To avoid potential loss of data, keep the program files within the program directory (this is because the program writes the files config.txt and data.csv and will overwrite any existing files with these names in the same directory).

  2. Do not modify the program files, including the program directory, in any way. For example, do not rename the program directory.

  3. The program saves the password of the new Gmail account you will create as plain text on your computer. This shouldn't be an issue as long as proper security measures are taken, although if you would like you can disable this feature by running the program with the correct option (see the README; note that this prevents the configuration file from being saved and will therefore require configuration of the program each time it is run).

Follow the steps below to get started:

Set up the Gmail Configuration

  1. Create a new Gmail account. This account should be used only to receive the workout summary emails forwarded from your usual account.

  2. On the Gmail account that typically receives the workout summary emails, set up a filter to forward all future mail from otbeatreport@orangetheoryfitness.com to the new Gmail account you just created. Since a Gmail filter cannot forward past mail, the data-extracting program has the ability to do this.

  3. In order for the data-extracting program to be able to read from the new Gmail account, you must disable two-factor authentication and enable "less secure app" access on the new account in the security settings for that account.

  4. (Optional) If you choose to forward past workout summary emails to your new account (in a later step), then you must temporarily disable two-factor authentication and enable "less secure app" access in the security settings of the account you wish to forward them from (that is, the account that typically receives the emails). Remember that the forwarding feature can currently only forward from Gmail accounts.

    Only do this if you are comfortable with the security risk this presents, and once you have finished running the program for the first time following the steps below, immediately revert the security settings back to normal by disabling "less secure app" access and re-enabling two-factor authentication for this account.

    You will only need to toggle these settings once for this account since the forwarding feature is only performed once by the program to forward all past emails. The Gmail filter you set up earlier will take care of forwarding all future emails.

Install the Program and Dependencies

  1. If you don't have it already, download the latest version of Python.

  2. Open a terminal and run the following on the command line:
    cd <install_directory> git clone https://github.com/JimmyVanHout/otf_data.git pip3 install flask pip3 install bs4
    where <install_directory> is the directory where you want to install the program. For example, if you plan to install the program in your home directory, the line would be:
    cd ~
    Note that in order to use the provided run script for future runs (Mac or Linux only), <install_directory> must be your home directory.

Run and Configure the Program

  1. Open a terminal and run the following on the command line:
    cd <program_directory_path> python3 otf_data.py
    where <program_directory_path> is the path to the program directory. For example, if you installed the program in your home directory, the first line would be:
    cd ~/otf_data

  2. When running the program for the first time, a configuration menu will appear.

    • The first checkbox asks whether you want to forward past emails to your new account. If you choose not to forward past emails, you can skip to the next bullet. If you do choose to forward past emails, fill out the section that pops up titled "One-Time Forwarding of All Past OT Beat Report Emails from This Account" with the following information:

      • Enter the email address of your usual Gmail account (the forwarding account).

      • Enter the password of your usual Gmail account.

      • Confirm the password of your usual Gmail account.

      • Enter the email address of your new Gmail account (the receiving account).

      • Leave the mailbox field blank (unless you receive the workout summary emails in a mailbox other than your inbox).

    • Fill out the section titled "Get OT Beat Report Data from This Account" with the following information:

      • Enter the email address of your new Gmail account.

      • Enter the password of your new Gmail account.

      • Confirm the password of your new Gmail account.

      • Under "Comma-separated originating email addresses", enter otbeatreport@orangetheoryfitness.com, (including the comma) followed by the address of your usual Gmail account.

      • Leave the mailbox field blank (unless you receive the workout summary emails in a mailbox other than your inbox).

    • Select the option to only search for new workout summary emails to make the program run faster in the future.

    • After filling out the information, submit the form.

The program will run and generate the data file, named data.csv, in the program directory, otf_data. Once the data file has been generated, upload it to the Data page of this site. The data will be analyzed and presented to you.

Note:

  • If any records contained errors, they will printed to standard output and not included in the file.

  • If you accidentally submitted incorrect information, simply delete the config.txt file in the program directory (otf_data) and then run the program again.

Next Time

Assuming you choose to leave two-factor authentication off and keep "less secure app" access turned on for your new Gmail account, in the future whenever you want to see your newest data on this site all you will need to do is listed below.

Example video: You can watch this example video of how to run the program after you have already run it once before.

  1. Running the program after the first time can be done in either of two ways (but only the first way if you use Windows), so choose one of the following:

    • Open a terminal and run the following on the command line (you will not have to complete the configuration again):
      cd <program_directory_path> python3 otf_data.py
      where <program_directory_path> is the path to the program directory. For example, if you installed the program in your home directory, the first line would be:
      cd ~/otf_data

    • Alternatively, use the provided run script (Mac or Linux only).

  2. Upload the data file to the Data page.

You may you choose to turn on two-factor authentication and turn off "less secure app" access after you run the data-extracting program, just note that you will need to change these settings whenever you want to run the program in the future.
